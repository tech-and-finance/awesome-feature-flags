# Contribution Guidelines

Thank you for contributing to Awesome Feature Flags!

## Adding a new item

- Search existing entries before adding a new one to avoid duplicates.
- Use the following format: `[Name](link) - Description.`
- Keep descriptions concise, factual, and non-promotional.
- End descriptions with a period.
- Add new items at the bottom of the relevant category.
- For managed services, mention the pricing model (free tier, per-seat, per-MAU, etc.) if notable.
- For open-source projects, link to the GitHub repo, not the marketing site.

## Quality standards

- The tool/resource should be actively maintained (commits in the last 12 months for software).
- Links should point to the canonical source (official repo or website).
- Articles should be from reputable sources and provide lasting value (not just news announcements).

## Pull request process

1. Make an individual PR for each suggestion.
2. Use imperative mood in the PR title (e.g., "Add Flipt" not "Added Flipt").
3. Include a short explanation of why the item is awesome.

## Reporting issues

Found a broken link or outdated entry? Open an issue describing the problem.
